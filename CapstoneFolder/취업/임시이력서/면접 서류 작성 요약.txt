1. 전문성: 지식, 기술에 포커스를 하자

2. 팀워크: 협업 과정에서 어떤 성향을 나타냈는지 = 태도

3. 노력한 경험: 태도

4. 지원 동기: ''질문이 없어도 저 안에 들어가도록 작성하자''
		너네 기업에 관심이 있다.
		예를 들어 3번에 도전적인 성격, 그것과 관련된 기업의 내용(너네 기업도 이렇게 하잖냐, 느그 기업이 비전을 이룰때까지 아세이 하겠다. 2~3줄만 써도 됨)

5. 입사 후 포부: ''질문이 없어도 쓰자''
	기술에 덧붙여 2줄 정도 작성, 뚜렷한 목표와 목표를 위해 어떤 노력을 할건지
-----------------------------------------------------------------------------------------------------------
Ex) 예상치 못한 문제를 해결했던 경험에 대해 작성하시오.
Ex) 협업 경험
Ex) 성격의 장단점
Ex) 최근 관심 이슈
Ex) 직무 전문성

최선은 이력서 1페이지, 자기소개서 1페이지
	내용이 길어지면 합쳐서 3페이지
	자소서 항목도 자유, 원하는 대로 설정(해시 태그로 나를 표현하거나.., 성격의 장점) -> 역량 어필
	
소제목에 끈기, 성실함 쓰면 식상함
 	사례 속에 있는 키워드가 제일임
		Ex) 끊임없는 브레인스토밍, 판매율 300% 향상
-----------------------------------------------------------------------------------------------------------
팁1: 자신이 살아온 인생 이야기와 실패 및 성공 등 특별한 경험이야기를 기술
어떤 기업인지 기술
GPT로 가독성 향상

ex) 지원동기
ex) 입사후의 포부

반복된 사례도 별로, 책임감(소통 능력)이 강하다는 표현도 별로
다양한 능력 X -> 대표적인 프로젝트 1개 언급하고 나머지들을 가볍게 언급
흔한 단어가 아닌 차별화된 단어를 언급하는 것이 중요하다(꼼꼼함은 별로임)


창의성을 의미하는 많은 패턴이 있음
	많은 언어 -> 글로벌 역량
	개방성 -> 글로벌 역량
아래와 같은 추상화를 쓰기 보다는 우리들만의 표현으로 차별화를 하자
얼마나 내 사례를 근거 있게 표현하고, 장점을 다른 사람과 다르게 표현하는 것이 중요
	그냥 책임감이 강했다고 하지 말고, 스스로에게 질문해 근거를 알아내자.

게임 모바일 클라이언트 개발자에게 필요한 지식, 기술, 태도를 각각 자세하게 알려줘

1. 지식
프로그래밍 언어:

C#: Unity 엔진을 사용한 개발에 필수적인 언어.
C++: Unreal Engine 및 성능 최적화에 유용.
Java/Kotlin: 안드로이드 네이티브 애플리케이션 개발에 필요.
Swift/Objective-C: iOS 네이티브 애플리케이션 개발에 필요.
게임 엔진:

Unity: 2D 및 3D 게임 개발에 널리 사용되는 엔진. 다양한 플랫폼에 배포 가능.
Unreal Engine: 고품질 그래픽과 복잡한 게임 로직에 적합한 엔진.
플랫폼 지식:

iOS 및 Android: 각각의 운영 체제에 대한 이해와 앱 배포 절차, UI/UX 가이드라인.
네트워킹:

TCP/IP, UDP: 네트워크 프로토콜에 대한 이해.
HTTP/HTTPS: RESTful API와 웹 통신.
데이터베이스:

SQLite, Realm: 모바일 환경에서의 로컬 데이터베이스.
Firebase: 클라우드 데이터베이스 및 실시간 데이터베이스 서비스.
게임 디자인 패턴:

Entity-Component-System (ECS): 게임 오브젝트 관리 패턴.
Singleton, Factory, Observer: 일반적인 디자인 패턴.
2. 기술
게임 개발:

물리 엔진: Unity나 Unreal에서 제공하는 물리 엔진 활용.
애니메이션: 캐릭터 및 오브젝트의 애니메이션 처리.
AI: 게임 내 적 또는 NPC의 인공지능 설계.
그래픽스:

Shader 프로그래밍: GPU를 활용한 그래픽스 처리.
3D 모델링 및 렌더링: Blender, Maya 등 도구 사용.
UI/UX:

사용자 인터페이스 디자인: 모바일 화면에 최적화된 UI 설계.
유저 경험(UX): 게임 흐름 및 사용자 경험 최적화.
성능 최적화:

프로파일링: 성능 문제를 진단하고 최적화.
메모리 관리: 제한된 모바일 환경에서의 효율적인 메모리 사용.
버전 관리 시스템:

Git: 코드 버전 관리와 협업 도구.
디버깅 및 테스트:

디버깅: Unity Debugger, Xcode Debugger 등.
유닛 테스트 및 통합 테스트: NUnit, XCTest 등.
3. 태도
문제 해결 능력: 복잡한 문제를 분석하고 해결하는 능력.
창의성: 새로운 게임 메커니즘과 아이디어를 창출하는 능력.
협업 능력: 팀원들과 원활하게 소통하고 협력하는 능력.
학습 열의: 빠르게 변화하는 기술 트렌드와 도구를 꾸준히 학습하려는 태도.
디테일에 대한 집중력: 작은 부분까지 신경 쓰는 세심함.
책임감: 프로젝트 일정과 품질에 대한 책임감.

이 모든 요소들이 조화를 이루어야 모바일 게임 클라이언트 개발자로서 성공적으로 일할 수 있습니다. 각 분야의 전문성을 키우고 지속적으로 발전해 나가는 것이 중요합니다.
